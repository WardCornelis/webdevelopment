Opdracht 3

Waarom staat de tekst in de blokken horizontaal gecentreerd?

Dit komt omdat alle div-elementen die op dezelfde rij(afhankelijk van je client resolutie) staan. Telkens op dezelfde hoogte beginnen (door de waarde inline-block), rekening te houden met de voorgaande rij. Ze zullen ze dus een rij vormen 1px lager dan het laagste element van de voorgaande rij. Als we dan ook weten dat tekst op een achtergrond, standaard links-boven geplaatst wordt. Dan krijgen we als resultaat, tekst dat horizontaal gecentreerd staat.

Ga na dat een property vertical-align:center geen effect heeft. Waarom eigenlijk niet?

Omdat de waarde center een ongeldige waarde is voor vertical-align.

Wat doet vertical-align dan wel?

Dit zorgt er voor dat de achtergronden in dezelfde rij(afhankelijk van je client resolutie) horizontaal gecentreerd worden.

Wat is de 'simpele' manier om een één enkele regel tekst verticaal te centreren?

Door het element line-height toe te voegen en deze dezelfde waarde te geven als de height voor de achtergrond. Dus als we de klasse 'tall' gebruiken zetten we deze op 150px, en indien we de klasse 'short' gebruiken. Zetten we deze op 50px. Standaard geven we het div-element een waarde van 100px mee.

Welke truc wordt vaak gebruikt om meerdere regels verticaal te centreren?

De tekst in het div-element plaatsen we in een p-element waardoor we andere parameters kunnen toepassen. Wanneer we dan bij het div-element display op table zetten. Kunnen we bij het p-element display op table-cell zetten. En vertical-align op middle. Hierdoor worden meerder regels tekst verticaal gecentreerd.

Waarom krijgt de regel voor class 'tall' voorrang?

Omdat de klasse 'tall' later voorkomt in het CSS bestand en dus de klasse 'area' overschrijft.
